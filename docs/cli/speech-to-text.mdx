## Get started

```bash
calibrate stt
```

The interactive UI guides you through the full evaluation process:

1. **Language selection** — pick from 10+ supported Indic languages
2. **Provider selection** — choose providers (only those supporting your language are shown)
3. **Input directory** — path to the directory containing your audio files and reference transcripts

The input directory should have this structure:

```
/path/to/data/
├── stt.csv
└── audios/
    ├── audio_1.wav
    └── audio_2.wav
```

The **stt.csv** file contains the reference transcriptions:

| id      | text                            |
| ------- | ------------------------------- |
| audio_1 | Hi                              |
| audio_2 | Madam, my name is Geeta Shankar |

<Note>
  All audio files should be in WAV format. The evaluation script expects files
  at `audios/<id>.wav` where `<id>` matches the `id` column in your CSV.
</Note>

Refer to the [sample dataset](https://github.com/ARTPARK-SAHAI-ORG/calibrate/blob/main/examples/stt/sample_input) for a template.

4. **Output directory** — where results will be saved (defaults to `./out`)
5. **API keys** — enter the API keys for the selected providers

The evaluation runs providers in parallel (max 2 at a time), showing the transcriptions as they are generated.

## Judge configuration

By default, a text LLM judge — routed through [OpenRouter](https://openrouter.ai/) (set `OPENROUTER_API_KEY` in your environment) — evaluates whether each transcription matches the reference text semantically. You can customize the judge model and add multiple evaluation criteria by passing an optional config file with `--config`:

```bash
calibrate stt -p deepgram google -i ./data -o ./out --config config.json
```

The config file supports:

```json
{
  "judge": {
    "model": "openai/gpt-4.1"
  },
  "evaluation_criteria": [
    {
      "name": "semantic_match",
      "description": "Check whether the values represented by both strings match semantically."
    },
    {
      "name": "completeness",
      "description": "All information from the source text is present in the transcription."
    }
  ]
}
```

| Key                   | Type   | Description                                                                 |
| --------------------- | ------ | --------------------------------------------------------------------------- |
| `judge.model`         | string | OpenRouter model id used by the LLM judge (default: `openai/gpt-4.1`). Use any model in the [OpenRouter catalog](https://openrouter.ai/models). |
| `evaluation_criteria` | array  | List of criteria to evaluate. Each criterion has `name` and `description`. |

Each criterion also accepts:

| Key         | Type    | Description                                                            |
| ----------- | ------- | ---------------------------------------------------------------------- |
| `type`      | string  | `"binary"` (default) or `"rating"`                                     |
| `scale_min` | integer | Required when `type` is `"rating"`. Lowest allowed score.              |
| `scale_max` | integer | Required when `type` is `"rating"`. Highest allowed score.             |

Binary criteria produce per-row pass/fail and a mean pass-rate. Rating criteria produce an integer score on your scale and a mean score in the leaderboard.

When multiple criteria are defined, each is scored independently and appears as a separate column in the results and leaderboard.

Refer to the [sample config](https://github.com/ARTPARK-SAHAI-ORG/calibrate/blob/main/examples/stt/config.json) for a template.

<Note>
  The `--config` flag is optional. When omitted, a single built-in criterion evaluates semantic match between source and transcription.
</Note>
